# Responsibilities

The ADR index helps contributors:

- Locate architectural decisions
- Understand framework evolution
- Navigate related documentation
- Trace historical design choices
- Identify dependencies between decisions

---

# Best Practices

- Keep the index synchronized with the ADR repository.
- Add every new ADR immediately after acceptance.
- Maintain numerical ordering.
- Do not remove superseded ADRs from the index.
- Update related categories as the framework evolves.

# Summary

The ADR Index provides a structured overview of every architectural decision in OpenMeta, serving as the primary reference for contributors, maintainers, and architects who need to understand the framework's design philosophy and evolution.

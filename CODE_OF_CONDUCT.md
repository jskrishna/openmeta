# Code of Conduct

---

# Purpose

OpenMeta is committed to fostering an open, inclusive, professional, and respectful community where everyone can collaborate, learn, and contribute.

This Code of Conduct establishes the standards of behavior expected from all community members and provides guidance for maintaining a welcoming environment for contributors of all experience levels.

---

# Our Commitment

As contributors, maintainers, and community members, we are committed to creating an environment that is:

- Welcoming
- Inclusive
- Respectful
- Collaborative
- Professional
- Constructive
- Safe

We believe that healthy collaboration leads to better software and a stronger open-source community.

---

# Scope

This Code of Conduct applies to all OpenMeta community spaces, including but not limited to:

- GitHub repositories
- Discussions
- Issues
- Pull Requests
- Documentation
- Community forums
- Chat platforms
- Social media interactions related to the project
- Public and private project events

It also applies whenever an individual is representing the OpenMeta project.

---

# Expected Behavior

Community members are expected to:

- Be respectful and professional.
- Welcome new contributors.
- Provide constructive feedback.
- Accept constructive criticism gracefully.
- Collaborate in good faith.
- Respect differing opinions and technical approaches.
- Focus discussions on improving the project.
- Help maintain a positive and productive environment.
- Credit the work of others appropriately.
- Respect project decisions once they have been made.

---

# Unacceptable Behavior

The following behaviors are not acceptable within the OpenMeta community:

- Harassment or discrimination of any kind.
- Personal attacks or insults.
- Trolling or inflammatory comments.
- Hate speech.
- Threats or intimidation.
- Deliberate misinformation.
- Publishing private or confidential information without permission.
- Spam or repeated off-topic discussions.
- Disruptive behavior that negatively affects collaboration.
- Any illegal or unethical activity.

---

# Communication Guidelines

When participating in discussions:

- Assume positive intent.
- Critique ideas rather than individuals.
- Explain technical decisions respectfully.
- Keep discussions focused and productive.
- Ask questions before making assumptions.
- Encourage evidence-based decision making.
- Remain patient with contributors of all skill levels.

---

# Contribution Standards

Contributors should:

- Follow the project's documentation.
- Respect the documented architecture.
- Follow coding standards.
- Submit focused and well-documented changes.
- Participate respectfully in code reviews.
- Accept review feedback professionally.
- Work collaboratively toward project goals.

---

# Maintainer Responsibilities

Project maintainers are responsible for:

- Enforcing this Code of Conduct fairly and consistently.
- Creating a welcoming environment.
- Encouraging respectful discussion.
- Resolving conflicts professionally.
- Protecting community members from harassment.
- Making decisions in the best interest of the project.
- Acting transparently whenever possible.

---

# Reporting Violations

If you experience or witness behavior that violates this Code of Conduct:

- Report the incident privately to the project maintainers.
- Include as much relevant information as possible.
- Allow maintainers sufficient time to investigate.
- Respect the privacy of everyone involved.

Reports will be handled confidentially whenever possible.

---

# Enforcement Process

Maintainers may take appropriate actions depending on the severity of a violation.

Possible actions include:

- Private warning
- Public warning
- Temporary suspension from community participation
- Removal of content
- Temporary restriction from contributing
- Permanent ban from project participation

The chosen action should be proportionate to the severity and frequency of the behavior.

---

# Enforcement Principles

Enforcement decisions should be:

- Fair
- Consistent
- Respectful
- Transparent where appropriate
- Focused on protecting the community

The goal is always to improve collaboration rather than punish individuals.

---

# Conflict Resolution

Whenever practical, maintainers should encourage constructive dialogue before applying enforcement measures.

Conflicts should be resolved by:

- Clarifying misunderstandings
- Encouraging respectful communication
- Seeking mutually acceptable solutions
- Prioritizing the health of the community

---

# Diversity and Inclusion

OpenMeta welcomes contributors regardless of:

- Experience level
- Background
- Nationality
- Language
- Age
- Disability
- Gender identity
- Sexual orientation
- Education
- Profession
- Technical expertise

Everyone deserves to participate in a respectful environment.

---

# Open Source Values

OpenMeta values:

- Transparency
- Collaboration
- Knowledge sharing
- Respect
- Continuous learning
- Technical excellence
- Constructive feedback
- Community-driven development

These values guide both technical and community decisions.

---

# Amendments

This Code of Conduct may evolve as the project grows.

Significant updates should:

- Be reviewed by project maintainers.
- Be communicated to the community.
- Preserve the project's commitment to openness and respect.

---

# Best Practices

- Be kind.
- Be respectful.
- Be professional.
- Be constructive.
- Listen before responding.
- Welcome newcomers.
- Keep discussions technical.
- Focus on improving the project.
- Respect community decisions.
- Help build a positive open-source culture.

---

# Summary

The OpenMeta Code of Conduct establishes a professional, respectful, and inclusive environment where contributors can collaborate effectively, share knowledge openly, and work together to build a high-quality open-source framework for the WordPress ecosystem.
